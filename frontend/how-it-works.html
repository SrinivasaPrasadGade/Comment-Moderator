The Journey of a Comment

Discover the architecture powering sub-millisecond AI moderation. From the frontend keystroke to deep-learning analysis.

1. The Frontend

A user types a comment in the sleek, minimalist chat interface. The moment they hit send, the payload is constructed instantly.

HTML5 Vanilla JS CSS Glassmorphism
socket.emit('send_message', {
  text: "This is amazing!",
  username: "user123"
});

2. WebSocket Bridge

Instead of slow HTTP requests, the comment travels over a persistent WebSocket connection, enabling real-time, bi-directional communication.

Socket.IO WebSockets

3. The Backend Engine

The Flask backend receives the event. It acts as the orchestrator, immediately handing the text over to the machine learning module before saving anything to the database.

Python Flask Flask-SocketIO
def handle_message(data):
  score = score_comment(data['text'])

4. ML Moderation Model

The text is tokenized and passed through Detoxify (a RoBERTa-based transformer model). It understands nuance, sarcasm, and context, identifying toxicity across multiple vectors (insult, threat, obscenity).

PyTorch Transformers Detoxify

5. Data Persistence

The comment, along with its ML-generated toxicity scores, latency, and flag status, is securely persisted to the relational database using parameterized queries to prevent injections.

SQLite Parameterized SQL

6. The Verdict

If safe, the message is broadcast to all users. If flagged, it is silently routed to the live Moderation Dashboard where human moderators can review the specific toxicity metrics and take action.

Event Routing Live Dashboard
